Spring’s JDBCTemplate

1. JdbcTemplate的初始化需要使用datasource.

可以在spring中定义一个datasource的bean.

<bean id="mySqlDataSource" class="org.apache.commons.dbcp.BasicDataSource">
<property name="driverClassName" value="com.mysql.jdbc.Driver"/>
<property name="url" value="jdbc:mysql://localhost:3306/JSDATA"/>
<property name="username" value="jsuser"/>
<property name="password" value="jsuser"/>
</bean>

2. 配置JdbcTemplate

  1. 实例化一个JdbcTemplate的对象,注入上面的datasource的bean.

    public class JdbcTemplateTest {
    private ApplicationContext ctx = null;
    private JdbcTemplate template = null;
    private DataSource datasource = null;
    public JdbcTemplateTest() {
    // Create a container forming the beans from datasource XML file
    ctx = new ClassPathXmlApplicationContext(“datasources-beans.xml”);
    // DataSource bean has been fetched from the container
    datasource = ctx.getBean(“mySqlDataSource”,DataSource.class);
    // Instantiate the template with the datasource
    template = new JdbcTemplate(datasource);
    }
    public static void main(String[] args) {
    JdbcTemplateTest t = new JdbcTemplateTest();
    // execute the data access methods from here
    ….
    }
    }

  2. 还可以直接把JdbcTemplate定义为一个bean,注入到DAO中.

3. 使用JdbcTemplate

通过使用JdbcTemplate提供的一系列方法,我们可以方便地把数据库的数据映射为Java程序中的域,比如String,int或者对象.

1. queryForObject, queryForList,queryForMap

template.queryForObject("select max(id) from TRADES", Integer.class);

该函数可以把查询的数据转换为Java的数据类型,集合.

2. 绑定查询条件中的变量

通过在查询条件中加入?(问号)占位符,在1中的函数重载加入一个Object[]数组,里面配置所需要绑定的变量即可.

public String getTradeStatus(int id){
String status =
template.queryForObject("select STATUS from TRADES where id= ?",
new Object[]{id}, String.class);
}
return status;

3. RowMapper(将查询结果映射为Java本地对象)

为了让JdbcTemplate把查询结果映射为java本地对象,你必须告知查询结果集和本地对象的关系,通过JdbcTemplate提供的RowMapper回调可以.

  1. 你可以直接继承它,并实例化后传入queryForObject函数.

    private static final class TradeMapper implements RowMapper{
    @Override
    public Trade mapRow(ResultSet rs, int rowNum) throws SQLException {
    Trade t = new Trade();
    // set the values by use ResultSet’s getXXX methods
    t.setId(rs.getInt(“ID”));
    ….
    return t;
    }
    }

    public Trade getMappedTrade(int id){
    Trade trade = template.queryForObject(“select * from TRADES where id = ?”,
    new Object[]{id} ,
    new TradeMapper());
    return trade;
    }

  2. 还可以使用匿名函数

    public Trade getTrade(int id){
    Trade trade = template.queryForObject(“select * from TRADES where id= ?”,
    new Object[]{id},
    new RowMapper(){
    @Override
    public Trade mapRow(ResultSet rs, int row) throws SQLException {
    Trade t = new Trade();
    t.setId(rs.getInt(“ID”));
    t.setAccount(rs.getString(“ACCOUNT”));
    ….
    t.setDirection(rs.getString(“DIRECTION”));
    return t;
    }
    });
    return trade;
    }

  3. 使用query方法将结果集映射为多个本地对象组成的List

将结果集映设为由基本数据类型组成的List,可以使用queryForList(),但想把结果集映设为一个有本地对象所组成的List,需要用到query方法和RowMapper

public List<Trade> getAllMappedTrades(){
List<Trade> trades =
template.query("select * from TRADES", new TradeMapper());
}
return trades;

4. 插入,删除,更新

  1. 插入

使用JdbcTemplate提供的update()方法

template.update("insert into TRADES values(?,?,?,?,?,?)",61,"JSDATA","REV",500000,"NEW","SELL");

  1. 如果需要绑定变量,可以使用?占位符

    template.update(“update TRADES set status=? where id=?”,status, id);
    template.update(“update TRADES set status=? where id=?”,new Object[]{status,id});

  2. 告诉JDBCTemplate绑定数据的sql类型,通过java.sql.Types

通过Object[]数组动态绑定的sql,如果Object里是String字符串而不是变量,需要告知 JDBCTemplate具体的sql类型

private void updateTradeUsingTypes() {
int rowsUpdated = template.update(
"update TRADES set status=? where id=?",
new Object[] { "UNKNOWN","6" },
new int[] { java.sql.Types.VARCHAR,java.sql.Types.INTEGER });
System.out.println("Rows Updated:" + rowsUpdated);
}

  1. 还可以调用存储过程

    template.update(
    “call JSDATA.REPLAY_TRADES_SP (?)”, tradeIds);
    }

5.execute()
可以运行DDL命令,诸如创建表,删除表

4.NamedParameterJdbcTemplate

使用 NamedParameterJdbcTemplate 可以让绑定变量的sql语句更具可读性.通过使用:name的形式绑定变量.有两种设置方法.

  1. 使用变量名作为键的Map

    public int getTradesCount(String s, String a){
    Map bindValues = new HashMap();
    bindValues.put(“status”, s);
    bindValues.put(“account”, a);
    int numOfTrades = template.queryForInt
    (“select count(*) from TRADES where account=:account and status=:status”,
    bindValues);
    System.out.println(“Number of Trades: “+numOfTrades);
    return numOfTrades;
    }

  2. 使用sqlParameterSource

通过该接口的addValue()方法

public int getTradesCountUsingSqlParameterSource(String s, String a){
SqlParameterSource bindValues =
new MapSqlParameterSource().addValue("status", s).addValue("account", a);
int numOfTrades = template.queryForInt
("select count(*) from TRADES
where account=:account and status=:status", bindValues);
...
}

还可以通过该方法绑定本地对象(JavaBean)的数据.

public int getTradesCountUsingBeanSqlParameterSource(Trade t){
SqlParameterSource bindValues = new BeanPropertySqlParameterSource(t) ;
int numOfTrades = template.queryForInt
("select count(*) from TRADES where account=:account and status=:status",
bindValues);
}

使用 NamedParameterJdbcTemplate 可以方便地使用数据绑定,但又是需要使用 JdbcTemplate 时,可以使用 NamedParameterJdbcTemplate提供的getJdbcOperation()方法.

5 Jdbc Batching

批处理可以重用语句,访问数据库请求也大大减少.

  1. 使用SqlParameterSourceUtils NamedParameterJdbcTemplate使用

SqlParameterSourceUtils会把众多对象的List转换为 SqlParameterSource的List.

private int[] insertTradesList(final List<Trade> trades) {
SqlParameterSource[] tradesList =
SqlParameterSourceUtils.createBatch(trades.toArray());
int[] updatesCount = namedTemplate.batchUpdate(
"insert into TRADES values
(:id,:account,:security,:quantity,:status,:direction)",
tradesList);
return updatesCount;
}

  1. 继承BatchPreparedStatementSetter

    private int[] insertTrades(final List trades) {
    int[] updatesCount = template.batchUpdate(
    “insert into TRADES values(?,?,?,?,?,?)”,
    new BatchPreparedStatementSetter() {
    @Override
    public void setValues(PreparedStatement ps, int i) throws SQLException {
    Trade t = trades.get(i);
    ps.setInt(1, t.getId());
    …
    }
    @Override
    public int getBatchSize() {
    return 10;
    }
    });
    return updatesCount;
    }

NamedParameterJdbcTemplate由于使用:占位符加上变量名的数据绑定方法使得
NamedParameterJdbcTemplate可以很容易的实现批处理,而对于 JdbcTemplate
使用?占位符而没有提供绑定变量的信息,只能通过位置来定义绑定的数据.
